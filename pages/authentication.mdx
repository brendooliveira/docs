<Tip>
  **Chave de API**: Sua credencial de acesso à API da Cpay. Esta chave identifica sua conta e autoriza suas requisições. **IMPORTANTE**: Sem a chave de API, as requisições serão recusadas.
</Tip> 

## <Icon icon="lock" type="solid"/>  Gerenciando chaves de API

Gerencie suas chaves de API diretamente em nossa plataforma. Você pode:
- Listar todas as chaves ativas
- Criar novas chaves
- Revogar chaves existentes

<Card title="Ambientes e Chaves de API" icon="book" horizontal>
  Todas as requisições são enviadas para o mesmo endpoint (`https://api.cpay.app.br`), mas o ambiente é determinado pela chave de API utilizada:
  - Chaves criadas em **dev** processam transações no ambiente de testes
  - Chaves criadas em **produção** processam transações reais
</Card>

<Card title="Erro de Autenticação" icon="warning" horizontal>
  A API retornará o código HTTP `401` quando:
  - A chave de API não for fornecida no header
  - A chave for inválida
  - A chave tiver sido revogada
</Card>

## <Icon icon="key" type="solid"/>  Criando chaves de API

<Card title="Boas Práticas de Segurança" icon="light-emergency-on" horizontal>
  * Armazene suas chaves em variáveis de ambiente ou gerenciadores de segredos
  * Nunca compartilhe suas chaves de API
  * Nós da Cpay nunca solicitará suas chaves
  * Revogue imediatamente qualquer chave comprometida
</Card>

Após criar sua chave, você pode começar a integrar com nossa API. Lembre-se de incluir a chave em todas as requisições no header `Authorization`:

```bash
curl -X POST https://api.cpay.app.br/v1/charge/create \
  -H "Authorization: Bearer sua_chave_api" \
  -H "Content-Type: application/json" \
  -d '{"amount": 1000, "name": "Brendo Oliveira", "document": "12345678901"}'
```

<Card title="Nota Importante" icon="info" horizontal>
  O mesmo endpoint (`https://api.cpai.app.br`) é usado tanto para o ambiente de desenvolvimento quanto para produção. O ambiente é determinado automaticamente pela chave de API utilizada na requisição.
</Card>
